About Fonts

note: when i say font, i technically mean typeface.  (this distinction doesn't matter with emoji fonts anyway).

these days, emojis are usually rendered from a font file. Nearly all fonts now use the OpenType (typical file extensions: .ttf/.otf/.woff/.woff2) format, but it has several different ways of storing color glyphs.  COLRv0 is the most widely supported of these, in most places.

Font Names

The name of a font is stored within the file, separate from the filename.  i.e: renaming the file won't do anything; you'll need to use a tool to change the strings in the name table.

Using

“Ok, but how do i actually make an application/website use this font?”

unfortunately, there isn't a simple answer to that.
most things will either the system emoji font, or they'll have a list of fonts that they check for.

basically, the process is different for each application and website and operating system.

Websites

many websites specify a list of emoji fonts.  they'll use the first one they find, and the list almost always begins with Apple Color Emoji, Segoe UI Emoji, ....
i.e:  i recommend installing a copy of Twemoji COLR with its name set to Apple Color Emoji (unless you are on MacOS).  This works on many websites, such as Bluesky, Github, etc.

Firefox

in about:config, set font.name-list.emoji to Twemoji COLR.  This will set the emoji font used in the UI and on any website which doesn't specify an emoji font.

Windows/MacOS

you may be able to install a copy of Twemoji COLR with the same name as the system emoji font (Segoe UI Emoji on Windows, or Apple Color Emoji on MacOS), to override it in applications which use the system font.
however, this can cause problems. (for example, if an application tries to render emojis without color: Segoe UI Emoji contains black-and-white versions of the emojis, while Twemoji COLR doesn't).
